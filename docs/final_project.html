29  Assignment: Tools-in-Action Book (Quarto + GitHub Pages)

This capstone presentation assignment asks students to demonstrate every tool covered in the course Sessions 1–19 (plus the pre‑Linux lecture) by authoring a Quarto “book” and publishing it on GitHub Pages. This document includes a structured outline, a repo scaffold, copy‑pasteable code examples for each tool, deliverables, and a point‑based rubric aligned to the course topics and notes. The relevant lecture pages are cited from the lecture notes (ywanglab.github.io)

Goal. Create a public Quarto book that teaches by doing: each chapter focuses on one tool you learned and includes explanations + runnable code examples that reproduce a tiny end‑to‑end workflow (data → features → tests → model → report). You’ll publish the book using GitHub Pages and tag a release (see lec 26 how to tag a release ).

Scope & coverage: Sessions 1–19 (Dev env, Git/GitHub, Quarto, RStudio cameo, Shell/Unix, Make, SQL I/II, pandas cleaning/joins/Parquet, rolling features, APIs, HTML scraping, pytest, PyTorch training loop) + Lecture 5 (Pre) Linux Basics. Use the course notes to refresh details. (ywanglab.github.io)

29.1 Learning outcomes

By completing this assignment, you will be able to:

  1. Demonstrate core DS productivity tools with concise, correct code examples.
  2. Assemble a Quarto book (book‑format site) and publish it from GitHub.
  3. Make a reproducible mini‑pipeline (data → SQL/joins → features → tests → model → report).
  4. Explain trade‑offs and best practices (ethics, leakage, CI, determinism).

29.2 Deliverables (what you must submit)

  1. Public GitHub repository named tools-in-action-<yourlastname> with:

    • book/ folder (Quarto project).
    • Reproducible code & data assets (small/synthetic OK).
    • requirements.txt (pinned), Makefile, and README.md with fresh‑clone steps.

  2. Published book URL (GitHub Pages) — place at top of README.

  3. Release tag v1.0-tools-in-action (Git tag pushed, see lec 26 for how to tag a version)

  4. Evidence the code runs: reports/ or _freeze/ artifacts (Quarto’s cached results), and a notebook that shows all the code and results.

29.6.4 05 — SQL with SQLite (Session 7) + 06 — Joins & Parquet with pandas (Sessions 9–10)

Create SQLite DB:

# scripts/make_sqlite.py (show in chapter)
import sqlite3, pandas as pd, pathlib
pathlib.Path("db").mkdir(exist_ok=True)
con = sqlite3.connect("db/prices.db")
pd.read_csv("data/processed/prices_with_vol.csv").to_sql("prices", con, if_exists="replace", index=False)

meta = pd.DataFrame({"ticker":["AAPL","MSFT"], "sector":["InfoTech","InfoTech"]})
meta.to_sql("meta", con, if_exists="replace", index=False)

# Simple inner join
q = """
SELECT p.ticker, p.date, p.adj_close, p.volume, m.sector
FROM prices p JOIN meta m USING(ticker)
ORDER BY date LIMIT 5;
"""
print(pd.read_sql(q, con))
con.close()

Equivalent pandas join + write Parquet:

import pandas as pd, pyarrow.parquet as pq, pyarrow as pa
prices = pd.read_csv("data/processed/prices_with_vol.csv")
meta = pd.DataFrame({"ticker":["AAPL","MSFT"], "sector":["InfoTech","InfoTech"]})
df = prices.merge(meta, on="ticker", how="left")
df["ticker"] = df["ticker"].astype("category")
table = pa.Table.from_pandas(df)
pq.write_table(table, "data/processed/prices.parquet")

29.6.5 07 — HTML Scraping & Parsing (Session 12)

Ethics: check robots.txt, throttle requests, and cache raw HTML. Then parse with BeautifulSoup.

import requests, time, pathlib
from bs4 import BeautifulSoup

URL = "https://example.com"  # keep it simple or save a local copy to data/raw/page.html
RAW = pathlib.Path("data/raw/page.html")

# Cache-first
if RAW.exists():
    html = RAW.read_text(encoding="utf-8")
else:
    # In practice: read robots.txt first; throttle; identify yourself.
    res = requests.get(URL, headers={"User-Agent":"STAT4160-student/1.0"}, timeout=10)
    res.raise_for_status()
    html = res.text
    RAW.write_text(html, encoding="utf-8")

soup = BeautifulSoup(html, "lxml")
title = soup.find("title").get_text(strip=True)
print("Page title:", title)

# Example: parse all links
links = [(a.get_text(strip=True), a.get("href")) for a in soup.select("a[href]")]
print("First 5 links:", links[:5])

29.6.7 09 — PyTorch: tiny training loop (Session 19)

# Minimal sequence regressor on synthetic returns
import torch, torch.nn as nn, numpy as np

# Make tiny dataset: (B,T,F) windows → next-step return
rng = np.random.default_rng(0)
T, F, N = 16, 3, 512
X = rng.normal(0, 0.5, size=(N, T, F)).astype("float32")
w = rng.normal(0, 0.2, size=(F,)).astype("float32")
y = (X[:, -1, :] @ w + rng.normal(0, 0.1, size=(N,))).astype("float32")

ds = torch.utils.data.TensorDataset(torch.from_numpy(X), torch.from_numpy(y))
dl = torch.utils.data.DataLoader(ds, batch_size=64, shuffle=True)

class SmallGRU(nn.Module):
    def __init__(self, F, H=32):
        super().__init__()
        self.gru = nn.GRU(F, H, batch_first=True)
        self.head = nn.Linear(H, 1)
    def forward(self, x):
        _, h = self.gru(x)
        return self.head(h[-1]).squeeze(-1)

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
net = SmallGRU(F).to(device)
opt = torch.optim.AdamW(net.parameters(), lr=3e-3)
loss_fn = nn.L1Loss()

for ep in range(1, 11):
    net.train(); tot=0
    for xb, yb in dl:
        xb, yb = xb.to(device), yb.to(device)
        opt.zero_grad(set_to_none=True)
        yhat = net(xb)
        loss = loss_fn(yhat, yb)
        loss.backward(); nn.utils.clip_grad_norm_(net.parameters(), 1.0)
        opt.step(); tot += loss.item()*xb.size(0)
    print(f"epoch {ep:02d} | train_mae={tot/len(ds):.4f}")

(Session 19 focus: Dataset/DataLoader + small training loop.) (ywanglab.github.io)
